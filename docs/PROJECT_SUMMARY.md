# 学生管理系统 - 项目总结

## 📋 项目概述

本项目是《软件工程》课程的实验项目，旨在设计并实现一个现代化的学生管理系统。项目采用前后端分离架构，严格遵循软件工程开发流程。

## 🎯 项目目标

- ✅ 提供完整的学生信息管理功能
- ✅ 实现基于角色的权限控制（RBAC）
- ✅ 支持多维度的数据管理（学籍、课程、成绩、考勤等）
- ✅ 提供直观易用的用户界面
- ✅ 遵循软件工程规范和最佳实践

## 🏗️ 技术架构

### 前端技术栈
- **框架**: Vue 3 - 渐进式 JavaScript 框架
- **构建工具**: Vite - 新一代前端构建工具
- **路由**: Vue Router 4 - 官方路由管理器
- **状态管理**: Pinia - Vue 3 状态管理库
- **UI组件**: Element Plus - Vue 3 组件库
- **HTTP客户端**: Axios - Promise based HTTP client

### 后端技术栈
- **语言**: Go 1.21+ - 高性能编译型语言
- **Web框架**: Gin - 高性能 HTTP Web 框架
- **ORM**: Gorm - Go 语言 ORM 库
- **数据库**: MySQL 9.2 - 关系型数据库
- **认证**: JWT - JSON Web Tokens
- **密码加密**: bcrypt - 密码哈希算法

### 架构特点
- **前后端分离**: 前端和后端独立开发、部署
- **RESTful API**: 标准化的 API 设计
- **单页应用（SPA）**: 流畅的用户体验
- **响应式设计**: 适配不同屏幕尺寸
- **模块化开发**: 清晰的代码结构

## 🔧 遇到的挑战与解决方案

### 1. CORS 跨域问题
**问题**: 前后端分离导致跨域请求被阻止
**解决**: 在后端添加 CORS 中间件，前端配置代理

### 2. JWT 认证流程
**问题**: Token 的存储和刷新机制
**解决**: 使用 localStorage 存储，请求拦截器自动添加

### 3. 数据模型关联
**问题**: Gorm 的关联关系配置
**解决**: 正确使用外键标签和 Preload 预加载

### 4. 前端状态管理
**问题**: 组件间数据共享
**解决**: 使用 Pinia 进行集中式状态管理
